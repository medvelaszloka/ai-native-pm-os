# Tool Selection Guide

When to use which AI tool for what.

---

## Quick Decision Matrix

| Task | Best Tool | Why |
|------|-----------|-----|
| File operations, commands, workflows | Claude Code | Direct system access |
| Long-form writing, research | Claude.ai | Better for extended conversations |
| Quick questions, brainstorming | Claude.ai | Faster for simple queries |
| Data analysis with files | Claude Code | Can read local files |
| Meeting transcription | Otter.ai / Fireflies | Purpose-built for audio |
| Image generation | Midjourney / DALL-E | Claude doesn't generate images |
| Spreadsheet automation | ChatGPT + Code Interpreter | Better for data manipulation |

---

## Claude Code vs Claude.ai

### Use Claude Code When:
- Working with local files (reading, writing, editing)
- Running slash commands (/today, /capture, /synthesize)
- Need terminal/bash operations
- Working within your Obsidian vault
- Automated workflows
- Multi-step file operations

### Use Claude.ai When:
- Extended brainstorming sessions
- Writing long documents without file context
- General research questions
- Complex reasoning without file access needed
- Sharing conversations with others

---

## Tool-Specific Tips

### Claude Code
- Keep conversations focused on one task
- Use slash commands for repetitive workflows
- Let it read files rather than pasting content
- Trust the tool outputs

### Claude.ai
- Start new conversations for new topics
- Use Projects for recurring contexts
- Attach files when needed
- Good for "thinking out loud" sessions

### ChatGPT
- Better for: Code interpreter, data viz, plugins
- Use when: Need specific integrations
- Avoid for: PM workflows (Claude understands Shape Up better)

### Perplexity
- Best for: Quick web research with sources
- Use when: Need current information
- Avoid for: Analysis tasks (better at retrieval than reasoning)

## Universal PM Use Cases for AI

| Use Case | What AI Does | Your Role |
|----------|--------------|-----------|
| Spec creation | Draft structure, fill sections | Judge quality, add context |
| Documentation | Auto-update post-deployment | Review accuracy |
| User research | Synthesize transcripts, find patterns | Interpret meaning |
| Metrics analysis | Pull data, create visualizations | Draw conclusions |
| Prototyping | Generate working demos | Define requirements |
| Content creation | Draft training materials | Ensure accuracy |

Source: [AI-Native PM Article](https://open.substack.com/pub/oliviaxmeng/p/i-cant-unsee-this-what-ai-native)

---

## When to NOT Use AI

- **Final decisions**: AI informs, you decide
- **Sensitive communications**: Review before sending
- **Legal/compliance**: Always verify with experts
- **Confidential data**: Be mindful of what you share
- **Relationship building**: Human touch matters
